Наследование - Inheritance.
Наследование - вид взаимоотношений между классами, при котором:
1 класс - это Потомок(Дочерний, Производный, Подкласс) - [Child, Derived, Subclass],
1 класс - перенимает все поля и методы у другого класса, но [ИСКЛЮЧАЯ: Приватные-private, конструкторы, статические методы и статические поля]

2 класс - это Предок(Родительский, Базовый, Суперкласс)[Parent, Base, Superclass]
+ А также: потомок может добавлять свои новые поля и методы!

Синтаксис одинаковый, как в С# и С++
class Parent{ поля и методы};
class Child : Parent { добавляет свои поля};

?Какая польза от наследования ?

1)Экономия времени и кода при написании классов на основе существующих. [иногда]
2)Наследование - создание "ПЛАТФОРМЫ" для выпуска различных сущностей, у которой есть базовый функционал.
!Как раз "ПЛАТФОРМА" - это есть Framework!
3)Платформа наследования позволяет провести быструю МОДЕРНИЗАЦИЮ существующих классов.
Меняем в платформе - а выгоду получают классы созданные на базе.
4)Во всех языках программирования (ЯП) потомок можно привести к типу предка. (работает неявное приведение). КИЛЛЕР-ФИЧА наследование.

Важно!
Существует ДВА вида наследования!
1)Одиночная - строго один ПРЕДОК(РОДИТЕЛЬ)
Такое работает в языках С#, Java, TypeScript, PHP
2)Множественное - более одного ПРЕДКА(РОДИТЕЛЯ)
Такое работает в языках С++, Python, ObjectPascal.

Когда говорят о наследовании, то говорят об "Дерево классов", выделяют три типа классов:
1)Корень (root) - класс, у которой нет Предков
2)Листья (Leafs) - класс, у которой нет потомков, но есть предки
3)Узлы (nodes)- класс, у которой есть и предки и потомки

В дереве есть 2 НАПРАВЛЕНИЯ:
1)От корня к листьям - "СПЕЦИАЛИЗАЦИЯ", чем дальше от корня тем классы более специфичные - specialization
2)Обратное - "ОБОБЩЕНИЕ" - GENERALIZATION


Также связано с наследованием - "ПОЛИМОРФИЗМ"
Есть несколько разных видов полиморфизма!
-При наследовании класс может иметь несколько методов с одинаковым именем, одинаковой сигнатурой, но разной реализацией.

Полиморфизм:
Это получается, что потомок имеет возможность "переопределить/заменить" унаследованный метод предка своей версией. (Главное, что сохраняется название метода, параметры, но работает по другому!)

В С# при наследовании есть 2 вида полиморфизма 
1)Ранее связывание - связь происходит при компиляции. - Быстро работает
2)Позднее связывание - связь происходит rin-time при выполнении кода: - чуть медленее работает
Для включения позденго связывания:
Два слова:
В классе предок для метода virtual 
В классе потомки для метода override

Разница между ними в скорости работе

В C#, Java, Python - есть особый (системный) класс - object (в С++ нет такого)
Особенность этого класса, что любой класс и любой тип (даже примитивные) автоматически являются ПОТОМКАМИ этого класса object.

Отличие Java от С# - что примитивные типы в Java не являются наследниками object, а только классы.

Из этого факта, есть ДВА полезных следствия:
Следствие 1)Любую переменную (любого типа) можно ПРИВЕСТИ к типу object (закон наследования - потомок можно привести к типу предка).
В C#, Java этот приём используются настолько часто, что спец. термином "boxing" - "упаковка"

??? Зачем это нужна "УПАКОВКА" ???
Существуют две основные причины (для упаковки) в программировании [такие же как в реальной жизни]
А)Хранение Б)Транспортировка
Т.е. Хранение - с помощью упаковки можно переменных разных ТИПОВ хранить в одной коллекции (например, в массиве).

---------------------------------

Дополнение к этой теме:
В C#, Java, Kotlin, PHP, TypeScript есть универсальные/обобщенные классы - Generics(дженерики) как в С++ Templates (шаблонные классы)

Generics (Дженерики) - "разворачиваются" и при компиляции и в runtime
В С++ Template (шаблонные классы) - "разворачиваются" только при компиляции.

Коллекции:
Основные коллекции (generics) в С#:
1)Список List<T>
Пример:
List<int> = list1 = new List<int>();

В C# - "разваорачиваются" в режиме компиляции и в режиме runtime, а в С++ в режиме компиляции

Для списка есть много полезных методов:
1).Add(переменная/значение); - Добавляется элемент в конец списка!
Пример: list1.Add(10);
Замечание: В отличии от массивов - список автоматически "растягивается"

2).Insert(переменная/значение); - вставка эл-та в любую позицию

3).Remove(значение); - Происходи поиск и удаляется первое встречное значение
4).Remove(индекс); - Удаляет по номеру индекса
5).RemoveRange(индекс, кол-во); это то же самое как и 3 - 4 пункты.
6).Sort(); - выполняется сортировка (модификация алгоритма Quicksort)
7).Index(значение); - линейный поиск первого вхождения значения.
8).BinarySearch(значение); -работает только если список сортирован
9).Count - количество элементов в списке.
